## 一、数据与方法

### 1.1 数据来源

- **原始表**：`behavior_features.csv`（按秒级时间窗 `window_sec` 与包名 `pkg` 聚合后的特征表）。
- **字段含义**：
  - `window_sec`：时间窗标识（秒级桶）。
  - `pkg`：Android 应用包名。
  - `event_cnt`：该窗内该包相关事件条数（活跃度）。
  - `uniq_cpu`：涉及 CPU 核心种类数。
  - `uniq_comm`：涉及线程名/命令名种类数（进程内复杂度）。
  - `uniq_tgid`：涉及进程（线程组）种类数（多进程协作强度）。

## 五、用户行为分析结论（可直接用于答辩）

1. **行为强度**：QQ 主包在多个时间窗出现显著高于基线（P90）的 `event_cnt`，可解释为页面切换或首屏加载链路中的高频内核事件（exec/binder 等）爆发。
2. **行为复杂度**：突发窗中 `uniq_comm` 明显升高，表明进程内多线程/多执行上下文参与增加，符合复杂页面或冷启动阶段特征。
3. **并发环境**：SystemUI 与 Launcher 的高事件量提示：解释 QQ 指标时必须说明 **系统 UI 同屏竞争** 属于正常现象。
